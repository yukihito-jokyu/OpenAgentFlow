# PR-Agent ガイド

## 概要

PR-Agent は、GitHub のプルリクエストに対して AI を活用したコードレビューや説明文生成を行うツールです。
本プロジェクトでは、PR コメントでコマンドを入力することで、各種 AI アシスタント機能を利用できます。

## ワークフロー設定

- **設定ファイル**: `.github/workflows/pr-agent.yaml`
- **トリガー**: PR 内のコメントが作成または編集された時
- **実行条件**: 
  - 送信者が Bot でないこと
  - コメントがプルリクエスト内で行われていること
  - コメントの先頭が `/` で始まっていること（コマンド実行の意思表示）

## コマンド一覧

PR のコメント欄で以下のコマンドを入力することで、各種機能を実行できます。

| コマンド | 内容・役割 |
|---------|-----------|
| `/review` | PR 全体のコードレビューを実行し、改善点や修正案を提示します。 |
| `/describe` | PR のタイトル、説明（Description）、変更内容の要約を自動生成します。 |
| `/improve` | コードの品質、可読性、パフォーマンスを向上させるための具体的な修正案を提示します。 |
| `/ask [質問]` | PR の内容について AI に質問できます（例：`/ask この変更による副作用はある？`）。 |
| `/reflect` | 変更内容に対する「なぜこの実装にしたのか」などの深い問いを投げかけ、思考を整理させます。 |
| `/generate_labels` | PR の内容を分析し、適切な GitHub ラベル（`bug`, `enhancement` など）を提案・付与します。 |

## 使用例

### 基本的なレビューを依頼する

```
/review
```

### 特定の質問をする

```
/ask この変更によるパフォーマンスへの影響はありますか？
```

### 複数コマンドの実行

複数の機能を利用したい場合は、別々のコメントでコマンドを実行してください。

```
/review
```

```
/describe
```

## 設定詳細

### モデル設定

| 設定項目 | 値 | 説明 |
|---------|---|------|
| `CONFIG.MODEL` | `openai/kimi-k2-instruct-0905` | 使用するメインモデル |
| `CONFIG.FALLBACK_MODELS` | 複数モデル | メインモデル失敗時のフォールバック |
| `CONFIG.CUSTOM_MODEL_MAX_TOKENS` | `128000` | コンテキストサイズ |
| `CONFIG.TIMEOUT` | `600` | タイムアウト（秒） |

### 出力のカスタマイズ

本プロジェクトでは、以下のカスタマイズを適用しています：

#### レビュー時 (`PR_REVIEWER.EXTRA_INSTRUCTIONS`)
- 日本語で回答
- 良い点も具体的に褒める
- 修正が必要な場合は理由と具体的な修正案を併記
- 深刻なバグ、パフォーマンス低下、セキュリティリスクを優先的にチェック

#### 説明文生成時 (`PR_DESCRIPTION.EXTRA_INSTRUCTIONS`)
- タイトル、サマリー、変更内容はすべて日本語で記述
- 箇条書きを活用し、技術に詳しくない人でも理解できる要約を含める
- 破壊的変更や依存関係の更新がある場合は特記事項として明記

## 必要なシークレット

GitHub リポジトリの Settings > Secrets and variables > Actions に以下のシークレットを登録する必要があります：

| シークレット名 | 説明 |
|---------------|------|
| `LITELLM_API_KEY` | LiteLLM Proxy 用の API キー |
| `LITELLM_PROXY_URL` | LiteLLM Proxy のエンドポイント URL（例: `https://example.com/v1`） |

> **Note**: `GITHUB_TOKEN` は GitHub により自動発行されるため、手動での設定は不要です。

## トラブルシューティング

### コマンドが反応しない

1. コメントが `/` で始まっているか確認
2. PR 内のコメントであることを確認（Issue のコメントでは動作しません）
3. Bot からのコメントでないことを確認

### タイムアウトエラー

大きな PR の場合、処理に時間がかかることがあります。現在のタイムアウト設定は 600 秒（10分）です。
それでもタイムアウトする場合は、PR を小さく分割することを検討してください。

### シークレット関連のエラー

- `LITELLM_API_KEY` が正しく設定されているか確認
- `LITELLM_PROXY_URL` が正しいエンドポイント URL か確認
- LiteLLM Proxy が正常に稼働しているか確認
